Stack Overflow Error

Java (Resolucao)

Inspecionar cuidadosamente os passos da pilha de erros e procurar o padrão repetitivo dos números de linha nos permite localizar a linha de código com as chamadas recursivas. Quando a linha for identificada, o código deve ser examinado e corrigido especificando uma condição de término adequada. Como exemplo, o traço da pilha de erros de uma instanciação pode se inspecionar:

A linha 4 pode-se ser vista repetindo, que é onde as chamadas recursivas são feitas e causando o erro.

Aumentar o tamanho da pilha de rollbar (-Xss)

Se o código tiver sido atualizado para implementar a recursão correta e o programa ainda lançar esse erro, o tamanho da pilha de rollbar pode ser aumentada para permitir um número maior de invocações. Aumentar o tamanho da pilha pode ser útil, por exemplo, quando o programa chama um grande número de métodos ou usa muitas variáveis locais.

O tamanho da pilha pode ser aumentado alterando o argumento -Xss na JVM, que pode ser definida ao iniciar a aplicação. Aqui está um exemplo: 

                        
-Xss4m

Isso definirá o tamanho da pilha do segmento para 4mb, o que deve impedir que a JVM jogue um java.lang.StackOverflowError.

Stack Overflow Error

Java (Mais Informações)

É um erro que ocorre em tempo de execução que aponta para problemas graves que não podem ser pegos por um aplicativo. Indica que a pilha de aplicativos está esgotada e geralmente é causada por recursão profunda ou infinita. 

                        
java.lang.StackOverflowError

O que pode causa esse erro em Java

Ocorre quando a pilha de aplicativos continua a crescer até atingir o limite máximo. Algumas das causas mais comuns para esse erro são:

  1. Recursão profunda ou infinita - Se um método se autodenomina recursivamente sem uma condição terminativa (Condição de parada).
  2. Relações cíclicas entre classes - Se uma classe instancia um objeto de uma outra classe, que por sua vez instancia um objeto da primeira classe. Isso pode ser considerado como uma forma de recursão. (Classe A que instância uma Classe B, onde está instância a Classe A).
  3. Aplicativos intensivos em memória - Aplicativos que dependem de objetos pesados de recursos, como documentos XML, classes GUI ou java2D.


Exemplo em Java quando ocorre esse erro

Aqui está um exemplo desse erro devido à recursão não intencional: (2º item de causas) 

                        
public class StackOverflowErrorExample {
    public void print(int myInt) {
        System.out.println(myInt);
        print(myInt);
    }

    public static void main(String[] args) {
        StackOverflowErrorExample soee = new StackOverflowErrorExample();
        soee.print(0);
    }
}

Neste exemplo, a recursiva chama-se várias vezes até atingir o tamanho máximo da stack, uma vez que uma condição de término não é fornecida para as chamadas recursivas. O tamanho máximo da pilha é atingido, e o programa sai com um :method print() java.lang.StackOverflowError
